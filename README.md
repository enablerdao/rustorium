## 🚀 クイックスタート

### 📦 インストール
```bash
curl -L https://get.rustorium.org | bash
```

### 🎮 ノードの起動

```bash
# シングルノード（最もシンプル）
rustorium --data-dir ./data

# 本番環境向け（ポート指定）
rustorium --data-dir ./data --port 4001

# 開発環境（複数ノード）
rustorium --dev --nodes 4

# 既存ネットワークへの参加
rustorium --bootstrap /ip4/127.0.0.1/tcp/4001
```

### 🔧 基本オプション
```bash
# 必須設定
--data-dir        # データディレクトリ (default: /tmp/rustorium)
--port            # P2Pポート (default: 4001)

# ネットワーク設定
--host            # ホストアドレス (default: 127.0.0.1)
--bootstrap       # ブートストラップノード
--external-addr   # 外部公開アドレス

# その他の設定
--keypair         # ノードの鍵ペアファイル
--log-level      # ログレベル (debug, info, warn, error)
```

### 🧪 開発モードオプション
```bash
--dev            # 開発モードを有効化
--nodes          # ノード数 (default: 4)
--base-port      # 開始ポート (default: 4001)
--no-browser     # ブラウザを自動で開かない
```

その他の詳細な設定は、Web UI または設定ファイル（`config/default.toml`）で変更できます。
```

### 🎛️ ノードの設定と管理

ノードの起動後、以下の方法で設定を変更できます：

#### 🌐 Web UI
ノード起動時に表示されるURLにアクセスし、Web UIから設定を変更できます：
- バリデーター設定
- シャード管理
- ネットワーク設定
- パフォーマンス設定
- メトリクス確認

#### 💻 CLI
ノード起動時に表示されるコマンドを使用して、CLIから設定を変更できます：
```bash
# ノードの状態確認
rustorium-cli status

# バリデーターとして設定
rustorium-cli validator enable --stake 100000

# シャードの作成
rustorium-cli shard create --name "my-shard"

# メトリクスの確認
rustorium-cli metrics

# ネットワーク設定の変更
rustorium-cli network set-peers --max 50

# パフォーマンス設定の変更
rustorium-cli perf set-limits --max-tps 10000
```
